# Competency Map

This page cross-references all role levels against the core competencies for each domain. Use your role column to identify your proficiency targets, then compare against your current level to build your learning plan.

**Proficiency scale:** P1 Awareness → P2 Foundational → P3 Practitioner → P4 Advanced → P5 Expert → P6 Leader

### Visual Insights Notes

- **P3 is the benchmark for independent delivery.** A Senior Analyst (L10) should be able to own a full report from data source to published workspace without supervision on any core competency.
- **BI Architecture Design** is explicitly a stretch target for the Specialist and above. Associates and Analysts are expected to understand existing patterns, not design new ones.
- **ALM / CI-CD** (Azure DevOps pipelines, Tabular Editor scripting, deployment pipelines) is increasingly required at L10 and a baseline expectation at L9.
- **Associate Manager** targets reflect delivery oversight more than hands-on depth — an AM needs to review, unblock, and set standards rather than build from scratch.

### Data Engineering Notes

- **SQL is a P2 baseline at Associate.** Any hire at L12 is expected to be able to write basic queries on day one; the P2 target at entry is intentional and non-negotiable.
- **Cloud platform proficiency** is domain-agnostic — the target is the skill (managing storage, compute, IAM, networking in a cloud context), not a specific vendor. Project context determines whether that is Azure, AWS, or GCP.
- **Data Governance** targets rise steeply from L9 because Specialists are the ones who set and enforce quality standards on their projects. Associates and Analysts execute within defined governance, they do not design it.
- **Data Architect / Lead Data Architect** P6 targets on Architecture Principles and Data Modeling reflect that these roles define the standards the rest of the practice follows.

## Reading the Map

1. Find your current role column.
2. For each competency row, note the target proficiency (Px).
3. Self-assess your current level honestly. A P3 target means "I can do this independently on a real project." If you need to check documentation for every step, you are probably P2.
4. Gaps between your self-assessment and the target are your learning priorities.
5. Use the role detail pages (under each domain in the left nav) for curated resource recommendations per competency.
