# Data Architect — Data Engineering

> **Career Level:** L9 Bridge | **Domain:** Data Engineering

## Overview

A Data Architect designs the technical blueprint for how data is structured, stored, integrated, and accessed across a client's data ecosystem. They translate business strategy into data architecture decisions — choosing platforms, defining integration patterns, modelling data domains, and ensuring the solution is scalable, secure, and governable. They operate across the full data stack and are the primary technical interface between business requirements and engineering implementation. This is a specialist IC (individual contributor) track for L9 engineers who progress through architecture depth rather than people management.

## Learning Goals

- Design and document enterprise-grade data architectures (Lakehouse, Data Vault, Data Mesh, Data Fabric)
- Produce architecture blueprints, technology roadmaps, and solution designs for client engagements
- Define physical and logical data models aligned to business domains
- Evaluate and select cloud data platforms and tools based on technical and commercial requirements
- Implement and govern metadata management, data lineage, and data catalog strategies
- Define data integration patterns (batch, streaming, event-driven, API-based)
- Apply data governance frameworks — roles, policies, data quality SLAs
- Contribute to pre-sales as a technical authority: solution scoping, effort estimation, architecture proposals
- Align data architecture decisions with security, privacy, and compliance requirements

## Key Skills

**Technical Skills** (from Data Architecture and Modern Data Management specializations)
- Data Architecture Principles
- Data Lakehouse Architecture
- Data Mesh and Data Fabric
- Data Vault Modeling
- Data Modeling Techniques and Methodologies (logical, physical, conceptual)
- Cloud Data Architecture (Azure, AWS, GCP)
- Modern Data Integration (batch, streaming, CDC, APIs)
- Database Architecture (relational, NoSQL, graph databases)
- Data Governance (policies, data ownership, metadata management)
- Data Lineage
- Knowledge Graph awareness
- Integration Architecture Blueprint and Design Solutions
- Technology Architecture Blueprint and Roadmap Definition
- Solution Architecture

**Soft Skills**
- Architecture communication — translating complex designs to non-technical audiences
- Whiteboarding and facilitated design workshops
- Risk identification and mitigation in architecture decisions
- Influencing without authority — aligning cross-functional teams to a shared design
- Written architecture documentation (ADRs, solution blueprints, data standards)

## Tools & Technologies

- **Architecture & modelling:** ERwin, draw.io, Lucidchart, Azure Architecture Center
- **Data platforms (full stack):** Databricks (Unity Catalog, Lakeflow), Snowflake (Polaris Catalog, Snowpipe), Azure Synapse Analytics, Amazon Redshift, Google BigQuery
- **Integration & streaming:** Apache Kafka, Amazon Kinesis, Azure Event Hubs, AWS Glue, Azure Data Factory
- **Governance & catalog:** Collibra Data Governance, Alation Data Catalog, AWS Glue Data Catalog, Databricks Unity Catalog
- **NoSQL & graph databases:** MongoDB, AWS Neptune, graph database patterns
- **Infrastructure as code:** Terraform (review/approval level)
- **Virtualisation:** Denodo Data Virtualization Platform
- **Data quality:** Ataccama, Collibra Data Quality and Observability
