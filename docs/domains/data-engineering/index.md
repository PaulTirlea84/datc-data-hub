# Data Engineering — Domain Overview

The **Data Engineering** domain covers back-end data infrastructure within DEVI. Practitioners in this domain construct, optimize, and maintain the infrastructure for data generation, transformation, and storage. They design pipelines, workflows, and data platforms that facilitate the seamless flow of data across systems — feeding the Visual Insights domain and client applications with clean, reliable, well-governed data.

## What Data Engineering Does

Data Engineering maps to four core Accenture specializations:

**Data Engineering** — construct and maintain infrastructure for data generation, transformation, and storage. Design pipelines and data platforms for seamless data flow. Skills: Big Data Architecture, Big Data Engineering, Data Engineering, and hands-on proficiency across the major cloud and open-source data tools.

**Data Architecture** — define and align the major types and sources of data necessary to support the business. Determine how data is used, stored, managed, and integrated within IT systems. Skills: Cloud Data Architecture, Data Architecture Principles, Data Lakehouse Architecture, Data Mesh, Data Modeling Techniques and Methodologies, Data Vault Modeling, Data Virtualization, Graph Databases, NoSQL, OLAP, Modern Data Integration, Database Architecture.

**Databricks** — design, build, and manage end-to-end data and AI workflows on the Databricks ecosystem, including clusters, pipelines, governance, and model lifecycle. Ensure scalable, secure, and optimized Lakehouse and AI deployment. Skills: Databricks Unified Data Analytics Platform, Databricks Unity Catalog, Databricks Lakeflow, Databricks Lakehouse Monitoring, Databricks MLFLOW, Databricks Mosaic AI, Databricks SQL, Data Lakehouse Architecture, Spark Declarative Pipelines, Microsoft Azure Databricks.

**Modern Data Management & Governance** — implement strategies for collecting, storing, processing, and governing data in complex modern data ecosystems. Ensure data quality, security, and accessibility. Skills: Data Governance, Data Governance Operating Model, Data Lineage, Data Mesh, Data Marketplace, Collibra Data Governance, Collibra Data Quality & Observability, Alation Data Catalog, Data Archiving.

## Key Tools and Technologies

- **Apache Spark** — large-scale distributed data processing
- **Apache Kafka** — real-time event streaming
- **Apache Airflow** — pipeline orchestration and scheduling
- **Snowflake Data Warehouse** — cloud-native analytical data warehouse
- **Databricks Unified Data Analytics Platform** — Lakehouse and AI workloads
- **Azure Synapse Analytics** — integrated analytics service on Azure
- **AWS Glue** — serverless ETL and data catalog
- **Amazon Redshift** — AWS data warehouse
- **Amazon EMR** — managed Hadoop and Spark clusters
- **Amazon Kinesis** — real-time data streaming on AWS
- **Google BigQuery** — serverless cloud data warehouse
- **MongoDB** — document-oriented NoSQL database
- **Terraform** — infrastructure as code for data platform provisioning
- **Docker** — containerized deployment of data services
- **Denodo Data Virtualization Platform** — data virtualization layer
- **IBM InfoSphere DataStage** — enterprise ETL
- **Collibra Data Governance** — data governance and catalog
- **Alation Data Catalog** — data discovery and stewardship
